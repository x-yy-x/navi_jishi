# 🏛 济事楼平面导览系统

同济大学软件学院济事楼（Jishi Building）室内平面导览系统，基于 **Fluid Navigation（流式导航）** 理念设计，提供流畅、直观的楼层导览体验。

---

## 项目概述

本项目是一个纯前端交互式室内地图导览系统，覆盖济事楼 **4F** 和 **5F** 两层，包含 **40+ 个房间**和 **25+ 个设施标记**（电梯、楼梯、卫生间）。系统以楼层平面图为底图，使用 SVG 叠加可交互的房间和设施区域，支持搜索、筛选、标定、详情查看等操作。

### 核心功能

| 功能               | 说明                                                     |
| ------------------ | -------------------------------------------------------- |
| **楼层切换** | 点击顶部标签栏或直接点击平面图上的电梯/楼梯图标切换楼层  |
| **房间搜索** | 支持按房间号、名称、类型、人员等多维度模糊搜索，加权排序 |
| **详情面板** | 点击房间显示简介、人员名单（含头像与职务）、实拍照片     |
| **标定模式** | 点击房间锁定视图，悬停其他房间不再弹出信息，再次点击取消 |
| **设施导航** | 电梯 🛗、楼梯 ⬆、卫生间 🚻 等地标设施一目了然           |
| **照片预览** | 每个房间配有实拍照片，支持多图展示，点击放大查看         |

## Fluid Navigation（流式导航）设计

本系统围绕 **Fluid Navigation** 理念构建，强调操作的连贯性和低认知负担：

### 1. 无缝的楼层流转

- 用户可通过三种方式切换楼层：顶部标签、搜索结果的楼层跳转、点击平面图上的电梯/楼梯
- 切换楼层时保留上下文状态（搜索栏清空、详情面板收起），避免信息混乱

### 2. 渐进式信息揭示

- **悬停** → 快速预览房间名称、类型
- **点击** → 展示完整详情（描述、人员、照片）
- **标定模式** → 锁定当前选中房间，方便仔细阅读
- 信息密度随用户的操作深度逐步增加，而非一次性铺开

### 3. 统一的交互范式

- 所有房间（矩形或多边形）使用相同的悬停高亮 + 点击选中的交互模式
- 设施标记（电梯、楼梯、卫生间）也支持矩形和多边形两种绘制方式
- 搜索下拉与详情面板风格统一，色彩编码系统贯穿始终

### 4. 容错与边界处理

- 搜索加防抖（150ms），避免频繁渲染
- 楼层切换加防抖（300ms），防止快速连点导致状态错乱
- 所有 DOM 操作带空值保护，单点错误不会拖垮整个页面
- SVG 渲染全流程 try-catch 包裹，异常可恢复


## 开发中遇到的问题与解决方案

### 1. 多边形房间的渲染与交互

**问题**：部分房间（如 443 综合事务管理中心、路牌与展示区）是不规则多边形，无法用简单的 `<rect>` 表示。

**解决**：在数据中增加 `shapeType`（`rect` / `polygon`）和 `points` 字段，渲染时根据类型分别创建 `<rect>` 或 `<polygon>` 元素。`getShapeCenter()` 和 `getShapeBounds()` 针对两种形状分别计算中心点和包围盒，确保标签位置和选中高亮准确。

### 2. 标注坐标与元数据的合并

**问题**：标注工具导出的 JSON 包含精确的像素坐标，而 data.js 包含房间类型、人员、描述等元数据。两者结构不同，手动合并极易出错且难以维护。

**解决**：编写程序化合并脚本，通过 `Function()` 构造器读取 data.js 中的 `buildingData` 结构，遍历 annotation JSON 按房间 ID 匹配，将标注坐标与元数据字段合并为最终的 `data.js`。

### 3. SVG 图片兼容性

**问题**：SVG 的 `<image>` 元素在不同浏览器中对 `href` 属性的支持不一致。

**解决**：同时设置 `href` 和 `xlink:href` 两种属性，确保跨浏览器兼容。

### 4. 竖屏平面图的适配

**问题**：楼层平面图（4F: 1045×1291, 5F: 901×1376）均为竖屏比例，而导览系统需要自适应容器宽度。

**解决**：将 SVG 的 `viewBox` 动态设置为图片尺寸，配合 `preserveAspectRatio="xMidYMid meet"`，让 SVG 按比例缩放适应容器。

### 5. 标定模式与悬停预览的冲突

**问题**：点击房间后鼠标移动时会不断触发其他房间的预览，干扰用户阅读。

**解决**：引入 `pinned` 状态变量。点击房间后进入标定模式，所有 `mouseenter`/`mouseleave` 事件根据 `pinned` 状态决定是否执行详情预览，再次点击或点击空白区域取消标定。

### 6. 电梯/楼梯切换楼层导致页面卡死

**问题**：快速连点电梯图标会导致楼层切换过程中状态错乱，偶尔出现页面空白卡死。

**解决**：三层防护——

- `_switching` 防抖标志（300ms 内禁止重复切换）
- `renderFloor` 整体包裹 try-catch，确保单点错误不中断渲染
- 所有 `document.getElementById` 调用增加 `null` 值判断

### 7. 搜索下拉菜单的子菜单被裁剪

**问题**：搜索悬停显示的人员子菜单因为父容器 `overflow-y: auto` 被裁剪，无法正确显示。

**解决**：将子菜单移出 `.search-results`，作为 `.search-wrapper` 的直接子元素，再通过绝对定位对齐到对应搜索项。

### 8. 设施标记的矩形 + 多边形双模式

**问题**：最初的标注工具只支持矩形绘制设施标记，但部分楼梯、厕所区域是不规则的。

**解决**：重构标注工具，让电梯、楼梯、卫生间、自定义标记同时支持矩形拖拽和多边形点击绘制，通过 `shapeType` 字段区分，与房间使用同一套渲染逻辑。

## 数据来源

- **平面图**：济事楼官方楼层平面图（PNG 格式）

## 快速开始

直接双击 `index.html` 在浏览器中打开即可使用，无需额外配置。

如需通过 HTTP 服务器运行（例如需要使用 Service Worker 或避免部分浏览器的本地文件限制）：

```bash
# 进入项目目录
cd navi

# 启动 HTTP 服务器（任选其一）
py -m http.server 3000
# 或
npx serve .
# 或
php -S localhost:3000

# 在浏览器中访问
# http://localhost:3000/index.html
```

> **注意**：绝大部分浏览器可直接通过 `file://` 协议打开使用。如遇图片加载异常，请使用 HTTP 服务器。
